# agents-md-craft — Craft & optimize AGENTS.md / CLAUDE.md

Make a project's agent-memory file (AGENTS.md / CLAUDE.md) lean, high-signal, and deliberately
hand-crafted, per HumanLayer's *Writing a good CLAUDE.md*. These files load into **every** session,
so every line must earn its place.

**You are an advisor/auditor, never an auto-generator.** Propose a draft/diff and a findings report;
**never write or overwrite any file until the user explicitly approves** (see the Review Gate — it is
the one hard, non-overridable rule). This skill is the counterweight to `/init`; do not auto-fill prose
the user has not confirmed, and do not invent unverifiable project facts.

Read [references/principles.md](references/principles.md) for the encoded HumanLayer principles
(versioned, so they can be updated) and [references/checklist.md](references/checklist.md) for the
audit/self-check checklist + the budget dashboard format.

## Modes

Detect the mode from the project and the request — the workflow is ~80% shared:
- **Optimize / audit** (most common) — a memory file already exists: audit, de-bloat, fix drift.
- **Create** — no memory file exists: build one from real, verified repo facts.
- **Staleness check** — re-verify an existing file's commands and `file:line` pointers still resolve.

Target **both AGENTS.md and CLAUDE.md** as cross-tool equivalents (AGENTS.md = emerging open standard
for Codex and others; CLAUDE.md = Claude Code's). Detect which exist, or ask which to target.

## Workflow

1. **Detect mode & targets.** Optimize vs create vs staleness; which of AGENTS.md / CLAUDE.md exist.
   For a tiny single-file project below real complexity, plan a single root file and **skip** the
   `agent_docs/` scaffold.
2. **Resolve the dual-file case.** If **both** files exist, do not maintain two copies — keep **one
   canonical file** (recommend the richer / more-recently-maintained; user overrides) and reduce the
   other to a thin **text pointer stub** ("Canonical instructions live in `<file>`"). Use a text
   pointer, **not a symlink** (cross-platform fragility). Produce two full independent files only on
   explicit opt-in.
3. **Inventory & classify.** Extract every existing instruction; tag each as WHAT / WHY / HOW and by
   universality (universal · task-specific · deterministic-tooling · project-context · command-recipe ·
   architectural-context · stale-risk). Flag non-universal, snippet, style-rule, attribution, and
   stale-risk content.
4. **Elicit the WHY.** Ask for purpose / component-role context that cannot be inferred from code.
   Surface unknowns as questions; never fabricate.
5. **Plan placement (root vs agent_docs/).** Keep in root only content that is **universal AND short**.
   Demote to `agent_docs/` anything **conditional, long, rarely-used, or reference/runbook/ADR** —
   **conditionality dominates size** (a short but niche rule still moves out). For universal-but-long
   content, keep a one-line summary + pointer in root and put detail in `agent_docs/`. Reference each
   `agent_docs/` file from root with a one-line "read when needed" description.
6. **Apply rewrites.** Replace style/formatting rules with a pointer to linters/formatters/pre-commit
   hooks ("never send an LLM to do a linter's job"). Replace copied code with **verified `file:line`
   pointers**. Remove attribution / "generated by" text.
7. **Run checks.** Produce the **budget dashboard** (green/yellow/red vs ~150–200 instructions / <300
   lines / <60 ideal — **overridable defaults**, cite HumanLayer provenance, account for the ~50
   instructions Claude Code's system prompt already uses). Verify markdown links and `file:line`
   pointers resolve. Walk the self-check checklist.
8. **Review Gate (HARD — never skip).** Present the proposed root file, any `agent_docs/` files, the
   stub, and the findings dashboard as a **draft/diff**, and **stop**. Require explicit user approval
   before writing anything. "Write" includes `agent_docs/` files and in-place edits, not just the root.
9. **Write on approval only.** Apply exactly what was approved.

## Guardrails

- Budgets and thresholds are **HumanLayer's opinions**, surfaced as overridable defaults — never
  hard pass/fail. The **only** non-overridable rule is the Review Gate (step 8).
- Stay scoped to agent-memory docs + their `agent_docs/` layer. Do **not** become a doc generator, a
  linter replacement, or a skill author — hand off SKILL.md authoring to `write-a-skill` / `skill-expert`.
- Keep this skill small: this file + the two references. See [references/checklist.md](references/checklist.md).
